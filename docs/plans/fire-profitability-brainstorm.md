## 0. 현재 상태 진단

### 파이프라인 퍼널

```
5,018 이벤트 (8일간)
  → 568 POS_STRONG (11.3%)
  → 50 LLM 판단 (1.0%)
  → 23 BUY (0.46%)
  → 12 close 데이터 있음
  → 2 승 / 8 패 / 2 무 = 승률 17% (raw)
```

v6 필터 적용 후: **6건, 3승/3패 = 50%, avg -0.02%, PF 0.93**

### 핵심 발견

| 발견 | 데이터 | 시사점 |
|------|--------|--------|
| **Peak는 t+5m~t+30m** | peak 14/23건이 5분~30분 | close까지 홀딩이 수익 파괴 |
| **SKIP 건 중 놓친 기회 0건** | 0/7 SKIP이 close>+0.5% | 필터링은 과하지 않음 |
| **UNKNOWN 52.5%** | 2,637/5,018 이벤트 | 절반 이상의 신호를 아예 분류 못함 |
| **BUY 시 중앙값 -0.16%** | close 기준 | 슬리피지+스프레드만으로 손실 |
| **Close 수익은 대부분 음수** | 8/12 패 | 뉴스 트레이딩은 본질적으로 단기 |

## 1. 현재 전략의 근본적 한계 (왜 50%에서 멈추는가)

### 1.1 구조적 한계: "뉴스 → 종가" 전략의 태생적 문제

**문제**: KRX 뉴스 공시 기반 day-trading은 "뉴스 반응 시간"이 핵심인데, 현재 시스템은:
- KIS API 폴링 주기 (3초) + 버킷 분류 + LLM 호출 (최대 12초) = **진입까지 ~15초**
- 이 시간 동안 이미 알고리즘 트레이더들이 가격을 움직임
- 대형주(ADV>1000억)는 뉴스 반영이 거의 실시간 → 우리가 사는 시점에 이미 반영 완료

**핵심**: 뉴스 트레이딩의 edge는 **"속도"**이지 **"분석 품질"**이 아님. LLM 기반 분석은 속도에서 열위.

### 1.2 TP/SL 비대칭

**현재**: TP=1.5%, SL=-1.0%
- TP 도달 빈도: 낮음 (12건 중 1건만 TP at t+5m)
- SL 도달 빈도: 높음 (3건)
- 대부분 close에서 -0.1%~-0.5% 구간에 착지 → TP도 SL도 안 터짐

**문제**: 뉴스 반응은 대부분 0~+1% 구간에서 끝남. TP 1.5%는 너무 높음.

### 1.3 시장 효율성 vs 진입 지연

대형주 공시 → 시장 반응 타임라인:
```
0s: DART 공시 게재
1-3s: HFT/알고 트레이더 매수 (0.5~1% 상승)
5-10s: 리테일 주문 시작
15s: Kindshot BUY 시그널 → 이미 0.5~1% 올라간 가격에 진입
30s-5m: 추가 모멘텀 (있으면 +0.5% 추가)
5m-30m: 차익 실현 시작 → 가격 하락
close: 대부분 원점 또는 소폭 하락
```

**Kindshot이 사는 가격 ≈ 이미 반영된 가격** → 평균 수익이 음수일 수밖에 없음.

### 1.4 샘플 부족

- 8일간 12건(close 있는)으로 통계적 유의성 없음
- 50% 승률은 동전 던지기와 구분 불가 (p-value > 0.5)
- 최소 50~100건의 거래가 있어야 전략 유효성 판단 가능

## 2. 수익성 양수 전환을 위한 구체적 아이디어

### 아이디어 1: Trailing Stop + 조기 이익 확보 (Impact: ★★★★★)

**현재 문제**: 고정 TP/SL은 뉴스 트레이딩에 비적합
**해결책**: 시간 기반 trailing stop

```
진입 후:
  0-30s: 관찰 (진입 직후 변동성 무시)
  30s-5m: trailing stop -0.3% (좁은 트레일)
  5m-30m: trailing stop -0.5% (약간 완화)
  30m-close: trailing stop -0.7% (더 완화)
  t+30m 이후: 무조건 청산 (장 마감까지 안 감)
```

**기대 효과**: peak(t+5m~t+30m) 포착률 대폭 상승
**구현 난이도**: ★★★ (가격 추적 루프 수정 필요)
**리스크**: 시뮬레이션 결과 없이 실행 위험

### 아이디어 2: 시간 기반 강제 청산 — "30분 룰" (Impact: ★★★★★)

**근거**: peak 14/23건이 t+5m~t+30m에 발생. close까지 홀딩은 가치 파괴.

```
규칙:
  - 진입 후 최대 30분 보유
  - t+30m 시점에 무조건 청산 (이익이든 손실이든)
  - SL은 유지 (-1.0%)
  - TP는 제거 (trailing stop으로 대체)
```

**시뮬레이션 (현재 데이터 기반)**:
현재 6건 중 t+30m 데이터가 있는 거래의 t+30m 수익:
- 알테오젠: +3.68% (t+5m) → 30분 내 TP
- HD현대중공업: +0.43% (t+30m) → 이익 확보
- HD한국조선해양: -0.55% (t+30m) → 소폭 손실
- 삼성물산: close에서 -0.09% → 30분 데이터 없음
- 삼성SDI: -0.96% (t+30m) → SL 터짐
- 서진시스템: -6.15% (t+30m) → SL 터짐

**기대 효과**: close까지 홀딩으로 인한 추가 손실 방지
**구현 난이도**: ★★ (price tracker에 timeout 추가)

### 아이디어 3: 소형주 집중 — "비효율 시장 공략" (Impact: ★★★★)

**근거**: 대형주(ADV>1000억)는 시장 효율성이 높아 뉴스 반영 즉시. 소형주(ADV 100~500억)는:
- 뉴스 반영 지연 (알고 커버리지 적음)
- 같은 계약이라도 시총 대비 임팩트 큼
- 스프레드가 넓을 수 있지만 수익 잠재력도 큼

```
전략 변경:
  - ADV 500억~2000억 구간에 집중 (현재는 500억 이상 무차별)
  - ADV > 5000억 대형주: confidence 상한 75 → 사실상 SKIP 유도
  - 계약금액/시총 비율 > 5%인 종목만 BUY
```

**기대 효과**: 시장 비효율 포착, 더 큰 수익 잠재력
**구현 난이도**: ★★ (config + prompt 조정)
**리스크**: 소형주 유동성 위험, 슬리피지 증가

### 아이디어 4: 확률 기반 진입 — "기다려서 사기" (Impact: ★★★)

**현재 문제**: 뉴스 감지 즉시 BUY → 고점 진입
**해결책**: 뉴스 감지 후 2-5분 관찰 → 가격 반응 확인 후 진입

```
Phase 1 (0-2분): 뉴스 감지, LLM 판단, 가격 관찰
Phase 2 (2-5분): 가격이 +0.5% 이상 → momentum 확인 → BUY
                  가격이 반응 없음 → SKIP (시장이 무시한 뉴스)
                  가격이 하락 → SKIP
```

**기대 효과**: "시장이 검증한 뉴스"에만 진입 → 승률 상승
**구현 난이도**: ★★★★ (비동기 관찰 루프, 가격 스트리밍 필요)
**리스크**: 좋은 진입점을 놓칠 수 있음

### 아이디어 5: UNKNOWN 마이닝 — "버려진 52%에서 보석 찾기" (Impact: ★★★)

**현재**: 5,018 이벤트 중 2,637건(52.5%)이 UNKNOWN → 완전 무시
**기회**: UNKNOWN 중에도 유의미한 신호가 있을 수 있음

```
파이프라인:
  1. UNKNOWN 배치 리뷰 (이미 구현: unknown_batch_review.py)
  2. LLM이 POS_STRONG으로 분류한 건 → 키워드 추출
  3. 반복 등장 키워드 → bucket.py에 자동 추가
  4. 점진적으로 UNKNOWN 비율 52% → 30% → 20% 감소
```

**기대 효과**: 거래 기회 증가 (현재 23건 → 40-50건으로)
**구현 난이도**: ★★ (이미 인프라 구축됨)
**리스크**: 노이즈 증가 가능

### 아이디어 6: 양방향 트레이딩 — "NEG_STRONG 공매도" (Impact: ★★★)

**현재**: NEG_STRONG 241건 → 전부 무시
**기회**: 부정 공시 후 하락 패턴은 상승 패턴보다 예측 가능

```
NEG_STRONG 공시 → SHORT 시그널:
  - 유증/CB 발행 → 공매도 (희석 확정)
  - 횡령/배임 → 공매도 (거버넌스 붕괴)
  - 대규모 소송 패소 → 공매도
```

**기대 효과**: 수익 기회 2배 (long + short)
**구현 난이도**: ★★★★ (KIS 공매도 API 연동 필요)
**리스크**: 공매도 규제 (KRX uptick rule), 개인 공매도 불가 이슈

### 아이디어 7: 멀티 타임프레임 — "스윙 홀딩" (Impact: ★★★)

**근거**: 진짜 좋은 촉매(FDA 승인, 대형 수주)는 하루 이상 효과 지속
**현재 제약**: `no_overnight=true` → 당일 청산 필수

```
전략 분리:
  Day-trade: 현재 전략 (대부분의 뉴스)
  Swing (1-3일): 고확신(conf>90) 대형 촉매만 오버나잇 보유
    - FDA 승인, 임상3상 성공
    - 매출 대비 20%+ 대형 수주
    - 기업가치 재평가급 M&A
```

**기대 효과**: 대형 촉매의 멀티데이 효과 포착
**구현 난이도**: ★★★ (보유 관리, 익일 모니터링 필요)
**리스크**: 오버나잇 갭 리스크, overnight margin

## 3. 파이어족 리스크 관리 전략

### 3.1 Kelly Criterion 기반 포지션 사이징

현재 승률/수익 구조 기반 Kelly fraction:

```
승률 p = 0.50
평균 승 W = 0.65% (TP 가중 평균)
평균 패 L = 0.70% (SL 가중 평균)
Kelly f* = p/L - (1-p)/W = 0.50/0.70 - 0.50/0.65 = -0.06

→ Kelly가 음수 = 현재 전략은 장기적으로 자금 소진
→ 수익성 양수 전환이 선결 과제
```

**목표 시나리오 (승률 60%, avg_win 1.0%, avg_loss 0.7%)**:
```
Kelly f* = 0.60/0.70 - 0.40/1.0 = 0.457
Half-Kelly = 22.8% → 총 자산의 22.8%를 1건에 투자
```

### 3.2 드로우다운 방어 규칙

```
Level 1 (일중 -1.5%): 포지션 사이즈 50% 축소
Level 2 (일중 -3.0%): 당일 신규 BUY 중단
Level 3 (주간 -5.0%): 1주일 거래 중단, 전략 리뷰
Level 4 (월간 -8.0%): 1개월 중단, 전략 근본 재설계
```

### 3.3 수익 곡선 모니터링

```
매주 체크:
  - 최근 20건 승률 > 55% → 정상 운영
  - 최근 20건 승률 40-55% → 사이즈 50% 축소
  - 최근 20건 승률 < 40% → 거래 중단, 원인 분석
```

## 4. 복리 성장 포지션 사이징 전략

### 4.1 Fixed Fractional (권장)

```
기본: 총 자산의 2% 리스크 per trade
예시: 자산 1억, 1건당 최대 손실 200만원
  → SL -1.0% 기준, 포지션 = 200만/1% = 2억 → 레버리지 필요
  → 실제 1x: 포지션 = 2000만원 (자산의 20%)
```

### 4.2 Anti-Martingale (승리 시 확대)

```
연속 2승 → 사이즈 1.5배
연속 3승 → 사이즈 2.0배 (상한)
1패 → 원래 사이즈로 복귀
```

### 4.3 복리 시뮬레이션 (목표 시나리오)

```
가정: 월 20거래일, 일 1건 평균, 승률 60%, avg_win +1.0%, avg_loss -0.7%
포지션: 자산의 20%

월간 기대 수익:
  = 20건 × (0.6 × 1.0% - 0.4 × 0.7%) × 20%
  = 20 × 0.32% × 20%
  = 1.28%/월

연간 복리:
  = (1 + 0.0128)^12 - 1 = 16.5%/년

→ 1억 시작 → 5년 후 2.14억 (연 16.5% 복리)
→ 파이어족 기준: 생활비 연 3000만 + 복리 16.5% = 자산 1.8억 이상 필요
```

## 5. 보조 수익원 아이디어

### 5.1 배당 + 뉴스 하이브리드

```
코어: 고배당 ETF (KODEX 고배당, TIGER 배당성장) 바이앤홀드

  - 배당 수익을 트레이딩 자본으로 재투자
  - 손실이 나도 배당으로 커버
```

### 5.2 ETF 모멘텀 스윙 (기 구현 중)

```
이미 크론 등록됨:
  - etf_shadow_trader.py (매일 16:00)
  - hybrid_strategy_v3.py (매일 16:08)

이 전략과 뉴스 트레이딩을 조합:
  - ETF 모멘텀: 중장기 트렌드 (1주~1개월 보유)
  - 뉴스 트레이딩: 단기 알파 (30분~당일)
```

### 5.3 옵션 프리미엄 (고급)

```
보유 주식에 covered call 매도:
  - 주가 상승 상한 제한되지만 프리미엄 수취
  - 횡보장에서 추가 수익

뉴스 트레이딩 수익 + 옵션 프리미엄 = 이중 수익원
(KRX 개인 옵션 제한 있으므로 ETF 옵션 활용)
```

### 5.4 자동화된 IPO/공모주 청약

```
KIS API로 IPO 정보 자동 수집
  → 공모가 대비 적정가 분석
  → 유망 종목 자동 청약
  → 상장일 매도 (평균 수익률 20-30%)
주 1-2회 발생, 승률 높음
```

## 7. 권장 실행 순서

### Phase A: 즉시 (1주)
1. **Trailing stop + 30분 룰** 구현 → 시뮬레이션
2. **소형주 집중** — ADV 기반 confidence 조정
3. 최소 20건 거래 데이터 수집

### Phase B: 2주차
4. **UNKNOWN 마이닝** — 배치 리뷰 실행, 키워드 확장
5. **성과 재측정** — 승률 60% 달성 여부 확인
6. Kelly/half-Kelly 포지션 사이징 도입

### Phase C: 1개월차
7. **확률 기반 진입** (시장 반응 확인 후 매수) 프로토타입
8. **스윙 전략** 고확신 건만 테스트
9. 드로우다운 방어 규칙 자동화

### Phase D: 분기
10. **보조 수익원** 통합 (ETF 모멘텀 + 뉴스 트레이딩)

12. 복리 성장 궤도 진입 확인

## 8. 핵심 메시지

> **현재 전략의 가장 큰 문제는 "뉴스를 분석하는 품질"이 아니라 "진입 속도"와 "청산 타이밍"이다.**
>
> LLM을 더 똑똑하게 만들어도 승률은 크게 오르지 않는다. 대신:
> 1. **더 빨리 나가라** (trailing stop + 30분 룰)
> 2. **비효율 시장을 공략하라** (소형주 집중)
> 3. **더 많은 기회를 만들라** (UNKNOWN 마이닝)
>
> 이 세 가지가 파이어족 수준의 지속 가능한 수익으로 가는 최단 경로다.
