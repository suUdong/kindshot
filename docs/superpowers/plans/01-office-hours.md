# Office Hours: Kindshot 제품 분석 및 방향 재정의

**날짜:** 2026-03-23
**프로젝트:** Kindshot — KRX 뉴스 기반 데이 트레이딩 MVP
**브랜치:** main
**모드:** Startup (1인 개발, pre-revenue)

---

## 1. 프로젝트 요약

Kindshot은 한국 주식시장(KRX)의 실시간 공시/뉴스를 LLM(Claude Haiku)으로 분석하여 BUY/SKIP 판단을 내리는 뉴스 기반 데이 트레이딩 시스템이다.

### 파이프라인 아키텍처
```
KIS API (뉴스/호가)
    |
    v
Feed Polling → Bucket 분류(키워드) → Quant 필터(ADV/spread) → LLM 판단 → Guardrails → Paper/Live 실행
    |                                                                              |
    v                                                                              v
UNKNOWN Review ← LLM 배치 분류                                      Price Tracker (TP/SL/Trailing Stop)
    |                                                                              |
    v                                                                              v
Keyword Feedback Loop                                               Replay 시뮬레이션 + 성과 리포트
```

### 핵심 수치
- **코드베이스:** ~9,500 LOC (28 Python 모듈)
- **테스트:** 27개 테스트 파일 존재, 현재 collection error로 대부분 실행 불가
- **버전:** 0.1.3
- **배포:** AWS Lightsail (systemd), Docker 지원 추가됨
- **모드:** Paper trading only (실거래 미실행)

## 2. 6가지 강제 질문 분석 (자율 진단)

### Q1: 수요 현실 (Demand Reality)

**현재 상태:** pre-product. 사용자 = 개발자 본인 1명. 외부 사용자 없음.

**진단:** 아직 "수요" 증거가 없다. Paper trading 모드에서 전략을 튜닝 중이며, 최근 커밋 히스토리를 보면 승률 17% -> 43% -> 50%로 개선 중이나, 이것이 실제 수익으로 전환된 적은 없다.

**핵심 질문:** 이 시스템이 내일 사라지면 누가 불편한가? — 현재로서는 아무도 불편하지 않다. 개발자 본인도 paper mode이므로 금전적 손실이 없다.

### Q2: 현재 대안 (Status Quo)

뉴스 기반 데이 트레이딩을 하려는 한국 개인 투자자의 현재 대안:
1. **수동 모니터링:** KIND/네이버 공시 직접 확인 + 수동 주문 — 느리지만 판단력이 있음
2. **기존 HTS/MTS 알림:** 키움/삼성증권 뉴스 알림 — 키워드 필터링은 가능하나 LLM 수준의 맥락 판단 불가
3. **유료 리서치/텔레그램 채널:** 전문가 의견 구독 — 인간 판단이므로 속도가 느림
4. **아무것도 안 함:** 대부분의 개인 투자자 — 뉴스 트레이딩 자체를 안 함

**진단:** "아무것도 안 함"이 가장 큰 경쟁자. 뉴스 기반 초단타 트레이딩은 기관/전문 트레이더 영역이며, 개인이 이 전략을 체계적으로 실행하는 것 자체가 드물다. 이것은 시장이 없다는 의미일 수도, 아직 도구가 없어서라는 의미일 수도 있다.

### Q3: 절박한 구체성 (Desperate Specificity)

**가장 필요로 하는 사람:** 한국 주식 데이 트레이딩을 시도하는 개인 투자자 중, 뉴스 속도에서 기관에 밀리고 있다고 느끼는 사람.

**문제:** 이 페르소나가 실제로 존재하는지, 그들이 "LLM이 공시를 읽고 BUY/SKIP을 알려주는 도구"에 돈을 낼 의향이 있는지 검증되지 않았다.

### Q4: 가장 좁은 쐐기 (Narrowest Wedge)

**현재 MVP의 wedge:** "KRX 공시 → LLM 분류 → 텔레그램 알림"

이것은 합리적인 wedge이다. 하지만 더 좁힐 수 있다:
- **최소 쐐기:** "특정 섹터(예: 바이오)의 공시만 모니터링하여 텔레그램으로 요약 전송" — 자동 매매 없이, 정보 제공만
- **이유:** 자동 매매는 규제/리스크/신뢰 장벽이 높다. 정보 제공은 장벽이 낮고 가치 검증이 빠르다.

### Q5: 관찰과 놀라움 (Observation & Surprise)

커밋 히스토리에서 관찰되는 놀라움들:
1. **승률 튜닝의 반복적 실패:** v1→v6까지 6번의 전략 튜닝. 키워드 확대, 프롬프트 강화, confidence 조정 등 시행착오가 많았다. → LLM 단독으로 트레이딩 판단을 내리는 것의 근본적 한계를 시사
2. **UNKNOWN 버킷이 핵심:** 분류되지 않는 뉴스가 많아 UNKNOWN 리뷰 파이프라인을 별도로 구축. → 키워드 기반 사전 분류의 한계
3. **소형주에서 더 나은 성과:** ADV 기반 필터가 소형주를 집중 타겟하도록 변경됨. → 대형주에서는 이미 기관이 선점

### Q6: 미래 적합성 (Future-Fit)

**현재 기술 스택의 5년 내 유효성:**
- LLM 비용은 계속 하락 → 유리
- KIS API는 안정적 → 유리
- 규제 리스크: 한국 금융당국의 AI 기반 자동매매 규제 가능성 → 위험
- 경쟁: 증권사/핀테크의 AI 뉴스 분석 내재화 → 장기적으로 독립 도구의 경쟁력 약화

## 3. 제품 방향 재정의

### 현재 포지셔닝의 문제
1. **자동 매매 시스템은 신뢰 장벽이 극도로 높다** — paper mode에서 벗어나지 못하는 것이 증거
2. **LLM 판단의 정확도가 불충분** — 6번의 전략 튜닝에도 PF(Profit Factor)가 1.12 수준
3. **1인 사용자 = 검증 불가** — 다른 사람이 써본 적이 없으므로 제품-시장 적합성 알 수 없음

### 제안: Pivot 검토

**Option A: 정보 서비스로 전환 (권장)**
- 자동 매매 제거, "AI 공시 분석 + 텔레그램 알림" 서비스로 전환
- 사용자가 판단, 시스템은 정보 제공
- 수익 모델: 구독 (월 5~10만원)
- 검증 방법: 텔레그램 채널 운영, 10명 무료 베타

**Option B: 백테스팅 플랫폼으로 전환**
- 이미 replay/collector가 잘 구축됨
- "당신의 뉴스 트레이딩 전략을 과거 데이터로 검증해보세요"
- 개발자/퀀트 타겟

**Option C: 현재 방향 유지 (Paper → Live 전환)**
- 가장 리스크 높음
- PF 1.5 이상, 3개월 paper 검증 후 소액 live 전환

### 권장: Option A
AI 비용 대비 정보 가치 제공이 가장 빠르게 검증 가능. 자동 매매의 규제/신뢰 장벽 없이 사용자를 확보할 수 있다.
