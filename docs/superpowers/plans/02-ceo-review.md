# CEO Review: Kindshot 10-Star Product 분석

**날짜:** 2026-03-23
**프로젝트:** Kindshot v0.1.3
**브랜치:** main
**모드:** SELECTIVE EXPANSION (현재 scope 유지 + 개선 기회 개별 제시)
**기반:** office-hours 분석 결과

---

## Step 0: Premise Challenge

### 이 제품이 풀고 있는 문제가 정말 존재하는가?

**현재 전제:** "개인 투자자는 공시/뉴스에 기관보다 느리게 반응하여 수익 기회를 놓친다. LLM이 이 갭을 메울 수 있다."

**도전:**
1. **속도 갭은 실재하는가?** — 그렇다. 기관은 알고리즘으로 공시 발표 후 수초 내 반응. 개인은 수분~수십분 후 반응. 이 갭은 실재한다.
2. **LLM이 이 갭을 메울 수 있는가?** — 부분적으로만. LLM 호출에 2-12초 소요. 기관 알고리즘 대비 여전히 느리다. 하지만 "뉴스를 읽고 맥락을 판단"하는 능력은 단순 키워드 알고리즘보다 우수하다.
3. **이 문제를 풀면 돈이 되는가?** — 불확실. PF 1.12는 수수료/슬리피지 제외 전 수치. 실거래에서 양수 수익 달성 여부 미검증.

**결론:** 문제는 실재하나, 현재 솔루션이 그 문제를 충분히 풀고 있는지는 미검증.

## 10-Star Product Framework

### 1-Star (현재): 뉴스 자동 분류 + Paper Trading
- 공시를 수집하고, 키워드로 분류하고, LLM이 BUY/SKIP 판단
- Paper mode에서만 작동
- 사용자 = 개발자 본인

### 5-Star: 신뢰할 수 있는 알림 서비스
- 텔레그램으로 실시간 공시 요약 + BUY 시그널 전달
- 일일 성과 리포트 자동 발송
- 10명 이상의 외부 사용자가 매일 확인
- 승률 55% 이상, PF 1.5 이상 달성

### 7-Star: 개인화된 트레이딩 어시스턴트
- 사용자별 관심 종목/섹터 설정
- 포트폴리오 맥락을 고려한 판단 (이미 보유 중이면 매수 차단 등)
- 웹 대시보드로 실시간 모니터링
- 히스토리컬 백테스팅 셀프서비스

### 10-Star: AI 트레이딩 코파일럿
- 자연어로 전략 설정 ("바이오 섹터 공시만, 신약 승인 관련만 알려줘")
- 실시간 리스크 관리 (자동 손절/익절 + 사용자 알림)
- 멀티 마켓 (KRX + NASDAQ + 암호화폐)
- 사회적 증거: 커뮤니티에서 전략 공유

## 핵심 리뷰 포인트

### 1. Focus as Subtraction — 지금 안 해야 할 것

현재 코드베이스에 이미 과도하게 많은 기능이 있다:

| 기능 | LOC | 핵심인가? | 판단 |
|------|-----|----------|------|
| Feed polling (KIS + KIND) | 470 | O | 유지 |
| Bucket 분류 | 498 | O | 유지 |
| Decision (LLM) | 308 | O | 유지 |
| Guardrails | 285 | O | 유지 |
| Price tracking | 365 | O | 유지 |
| Replay | 1,227 | ? | **과도** — 검증 도구이나 제품 코어 아님 |
| Collector | 1,235 | ? | **과도** — 백필은 나중에 |
| Unknown Review | 1,068 | ? | **과도** — 키워드 자동 확장은 최적화 단계 |
| KIS Client | 636 | O | 유지 (API 래퍼) |

**진단:** 9,500 LOC 중 ~3,500 LOC (37%)가 "있으면 좋지만 지금 핵심이 아닌" 기능. 1인 개발에서 이 코드를 유지보수하는 부담이 핵심 기능 개선 속도를 늦추고 있을 가능성.

**권장:** Replay/Collector/Unknown Review는 현재 상태 동결. 핵심 파이프라인(Feed → Bucket → Decision → Guardrail → Alert) 품질에 집중.

### 2. Speed Calibration — 빠르게 결정해야 할 것

**Two-way door (빠르게 결정):**
- 텔레그램 알림 강화 → 바로 실행 가능, 되돌리기 쉬움
- Paper → 소액 Live 전환 테스트 → 10만원 규모면 리스크 낮음
- 외부 사용자 3명 모집 → 행동이지 코드가 아님

**One-way door (신중하게 결정):**
- 유료 서비스 전환 → 법적/규제 검토 필요
- 실거래 자동 주문 실행 → 금융사고 위험
- KIS Live API 키 발급 → 증권사 심사 필요

### 3. Inversion Reflex — 실패 시나리오

**이 프로젝트가 실패하는 가장 가능성 높은 경로:**
1. **전략이 수렴하지 않음** — 6번 튜닝했지만 PF가 1.0~1.2 사이를 맴돌고, 실거래에서는 슬리피지로 손실 전환
2. **개발자 번아웃** — 혼자 9,500 LOC를 유지보수하면서 전략 튜닝까지 하는 것은 지속 불가능
3. **규제 리스크** — 한국에서 AI 기반 자동매매에 대한 규제가 강화되면 사업 자체가 불가능

**가장 과소평가된 리스크:** #2. 코드베이스가 이미 1인이 관리하기엔 크다.

### 4. Proxy Skepticism — 잘못된 지표를 쫓고 있는가?

**현재 추적 지표:** 승률, PF, 일일 P&L (paper)
**문제:** 이 지표들은 paper trading에서만 유효. 실거래 환경에서는:
- 슬리피지로 실제 체결가가 다름
- 호가 유동성이 paper에서 가정한 것보다 나쁨
- LLM 지연 시간이 수익에 직접 영향

**더 나은 지표:**
- "LLM 판단 후 실제 가격 변동"과 "판단 시점의 가격" 차이 → 시그널의 진짜 가치 측정
- "외부 사용자가 알림을 보고 행동한 비율" → 정보 가치의 직접 측정

## Verdict

**Kindshot은 흥미로운 기술 프로토타입이지만 아직 제품이 아니다.**

핵심 과제는 코드를 더 쓰는 것이 아니라:
1. 전략이 실거래에서 양수 수익을 내는지 검증
2. 개발자 본인 외에 누군가가 이 정보를 가치있다고 느끼는지 검증

코드는 충분하다. 이제 사람과 시장으로 나가야 할 때.
