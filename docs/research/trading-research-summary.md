# 트레이딩 리서치 요약 (2026-03-12)

## 핵심 수치
- 왕복 거래비용: 0.45~1.0% (수수료+세금+스프레드+슬리피지)
- 통계적 유의성: 최소 100건, 이상적 200~500건
- 수익 데이트레이더 비율: 1~3% (학술 연구 기준)

## 절대 하지 말 것
- 첫날 3승 근거로 룰 수정
- 시장가 체결을 paper fill 기본값으로
- 키워드 하나로 체결/해지 구분 없이 같은 이벤트 취급
- 장전/장중/장마감을 같은 전략으로
- kill switch 없이 live 전환

## 버킷 튜닝 (Claude + ChatGPT 리서치 종합)
- 유상증자/CB발행/공급계약 해지 → NEG_STRONG 확정
- 자사주 소각(이익소각) → POS_STRONG, 자사주 매입(주가안정) → POS_WEAK, 임직원보상 → NEG
- 공급계약 체결: 매출 대비 15% 이상, **당일용 신호** (다음날 반전 연구결과)
- 특허: **KOSDAQ만 유효**, KOSPI 특허는 통계적으로 유의하지 않음
- 체결/해지, 판매/구매 구분 필수 ("공급계약" 키워드만으로는 부족)
- source 구분: 공식공시(DART/KIND) vs 언론기사 분리 필요

## 버킷별 보유시간 가설
- 공급계약: 1~15분 (당일 반전 리스크)
- 주주환원(소각): 15분~EOD
- 특허/임상: 5~30분 (KOSDAQ만)

## 시간대 특성
- 장전 공시(06:00~08:30): 개장 갭 가장 큼
- 09:00~09:30: 변동성 최고
- 11:00~14:00: 비유동, 스프레드 필터 강화 필요
- 15:20~15:30: 마감 동시호가

## 실전 전환 체크리스트
1. Phase 1: 페이퍼 4~8주, 100건+ 거래
2. Phase 2: 마이크로 라이브 (1주, 최소수량)
3. Phase 3: 소규모 (목표의 10%)
4. Phase 4: 2주마다 25%씩 확대
- 페이퍼 성과의 50~70%를 실전 기대치로

## 성과 기준 (1개월 후)
- Profit Factor > 1.5
- Expectancy > 0
- Sharpe > 1.0
- 미달 시 시스템 재설계

## 참고 교훈
- LLM은 자율 판단자가 아닌 증폭기로 사용
- 설명 불가능한 전략 = 과적합 의심
- 장중 공시는 당일 내 가격 완전 반영 (보유 길수록 알파 소멸)
