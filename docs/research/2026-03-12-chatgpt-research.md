# ChatGPT Deep Research: 뉴스 기반 자동 단타 시스템 생존 가이드

> 2026-03-12 ChatGPT Deep Research로 생성

## 1. 초보가 빠지기 쉬운 함정

### 1-1. 뉴스 기반 트레이딩에서 흔한 실패 패턴

"좋은 뉴스"가 아니라 "이미 가격에 반영된 뉴스"를 사는 것이 대표적 실패. 한국 연구에서 단일판매·공급계약 공시는 공시 당일 양(+) 반응이 있었지만 다음 날 반전이 관찰됐고, KOSPI 특허공시 연구에선 공시 1분 뒤 평균 누적초과수익률 0.92%가 나타났다. 공정공시 연구도 가장 큰 반응이 공시 시점 0분, 대부분이 1분 내에 발생했다고 봤다.

**액션:**
- 모든 이벤트에 source_original_ts, feed_receive_ts, bucket_ts, llm_decision_ts, order_send_ts, first_fill_ts 저장
- 원문시각 대비 가격 선반영, VI 여부, 스프레드 급확대, 이미 1~2% 이상 급등을 자동 SKIP 조건
- 공식공시와 언론기사 버킷 분리
- 중복 기사 fingerprint로 같은 내용 여러 번 사지 않게

**주의:** "공급계약" 키워드 하나로 POS_STRONG 처리하면 체결/해지, 판매/구매를 섞어버릴 위험. KRX 해설서상 회사의 매출에 반영되는 수주·공급계약만 공시대상이며 발주·구매계약은 원칙적으로 공시대상이 아님.

### 1-2. 백테스트에서는 잘 되는데 실전에서 안 되는 이유

Bailey 등의 연구: 같은 데이터에 반복 테스트하면 몇 백 번만 돌려도 인샘플에서 매우 좋아 보이는 전략을 쉽게 만들 수 있고, 아웃오브샘플 성과는 실망스러운 경우가 많다.

**액션:**
- 지금 룰을 2주간 freeze, 버킷/프롬프트/threshold 수정 없이 로그만 쌓기
- 실험 장부 만들기: 프롬프트 한 줄 수정도 1 trial로 카운트
- 체결은 3개 모델로: 낙관(다음 mid), 현실(다음 ask/bid), 보수(다음 ask/bid + 1틱 불리하게)
- "LLM 문구만 조금 손본 것"을 전략 변경이 아니라고 생각하면 안 됨

### 1-3. 수수료·세금·슬리피지

한국 주식 단타 비용:
- KOSPI 매도: 증권거래세 0.05% + 농어촌특별세 0.15% = 0.20%
- KOSDAQ 매도: 0.20%
- 키움 HTS 수수료: 0.015%
- 왕복 명시 비용: ~0.23%, 슬리피지 포함 시 0.3%+

**액션:**
- 손익식: gross pnl - buy fee - sell fee - sell tax - entry slippage - exit slippage
- 뉴스 급등주에선 시장가 주문 금지 기본값
- KOSPI 특허공시 초단기 연구: 시장가 매수·매도 전략은 음(-) 수익률

### 1-4. "3건 전부 수익" 해석

3건 평균 gross 수익 약 1.82%. 명시 비용 0.23% 차감해도 플러스이지만, 표본 3건은 "오늘은 잘 됐다" 정도만 말할 수 있음.

**액션:**
- 최소 2~4주간 룰 freeze 후 관측만
- 성과표를 기대값 = 승률×평균이익 - 패배율×평균손실 - 비용 중심으로
- SKIP한 종목의 이후 1분/5분/15분/EOD 수익률도 저장
- 버킷별 30~50건 전에는 해석만, 100건 전에는 임계값 수정 금지

## 2. 한국 주식시장 특수성

### 2-1. 한국 공시 구조

OPEN DART가 공시 원문과 API 제공, KIND/KRX가 기업공시 채널과 실시간 공시 웹 서비스 제공.

**액션:** 버킷 분류에 추가할 필드:
- source_system (DART/KIND/언론)
- form_type (투자판단관련 주요경영사항/단일판매·공급계약/자기주식취득/주식소각 등)
- direction (체결/해지/정정)
- materiality_ratio (매출액·자기자본 대비)
- treasury_purpose (주가안정/소각/임직원보상 등)

**주의:** 자기주식은 적대적 기업매수 방어, 임직원 공로금, 주가 안정 등 서로 다른 목적. "자사주"라는 단어 하나로 같은 호재로 보면 안 됨.

### 2-2. 공시 유형별 단타 기대값

**우선 관찰 가치 높은 이벤트:**
- 자사주 소각: 매입보다 소각 공시 이후 주가가 더 증가
- 주가안정/이익소각 목적 자기주식 취득: 양(+) 초과수익률 (임직원 인센티브 목적은 음(-))
- 대형 단일판매·공급계약 체결: 공시 당일 양(+) 반응, **다음날 반전**. "당일용 신호"
- 실적 surprise: 시장기대치 초과/미달에 따라 반응 (하락장에선 효과 약화)
- KOSDAQ 특허/기술 공시: 유의한 양(+), **KOSPI는 통계적으로 유의하지 않음**

**기본 제외 후보:**
- 유상증자/희석형: 할인율 클수록 더 부정적, 비정상 공매도 증가
- 임직원 보상 목적 자기주식: 평균적으로 부정적
- 공급계약 해지/정정

**버킷 재분류 초깃값:**
- POS-A: 주식소각, 주가안정/이익소각 목적 자사주 취득, 대형 공급계약 체결, 기대치 상회 실적
- POS-B: 중요 특허/기술/임상/무형자산 관련 투자판단 관련 주요경영사항
- NEG/EXCLUDE: 유상증자, 일반공모, 큰 할인발행, 공급계약 해지, 임직원 보상 목적 자사주

**버킷마다 기본 보유시간을 다르게:** 공급계약 1~15분, 주주환원형 15분~EOD

### 2-3. 거래 비용 구조

호가단위: 2,000원 미만 1원, 2,000~5,000원 5원, 5,000~20,000원 10원, 20,000~50,000원 50원, 50,000~200,000원 100원, 200,000~500,000원 500원, 500,000원 이상 1,000원.

정규시장: 09:00~15:30, 가격제한폭: 기준가 대비 ±30%.

**액션:**
- 모든 지표를 %와 tick 둘 다 저장
- 예상 gross edge < 명시 비용×2 + 예상 슬리피지이면 진입 금지

### 2-4. 시간대별 특성

- 장전 호가접수: 08:30~09:00
- 장전 시간외종가매매: 08:30~08:40
- 정규시장: 09:00~15:30
- 장후 시간외종가매매: 15:40~16:00
- 시간외단일가: 16:00~18:00
- NXT: 프리마켓 08:00~08:50, 메인마켓 09:00:30~15:20, 애프터마켓 15:40~20:00

**액션:** 분석을 최소 6구간으로: 08:30~09:00, 09:00~09:10, 09:10~14:30, 14:30~15:20, 15:20~15:30, 15:30 이후

### 2-5. 외국인·기관 수급

외국인/기관 수급은 신호 자체라기보다 시장 상태(confirmer)에 가깝다. 방향성은 있어도 단순 추종 규칙은 불안정.

**액션:**
- same_day_foreign_flow, same_day_institution_flow를 confidence scaler로만 사용
- 긍정 이벤트 + 외국인/기관 순매수 강세면 size 1.0, 아니면 size 0.5

## 3. 리스크 관리 기초

### 3-1. 하루 손실 한도

- 1회 트레이드 리스크: 계좌 기준 0.25% 이하
- 하루 손실 한도: 계좌 -1.0% 또는 -3R 중 먼저 도달한 값
- 한도 도달 시: 당일 신규 진입 중단, 로그만 수집
- 사람 재량 예외 금지

### 3-2. 연속 손실 대응

- 같은 버킷 2연패: 해당 버킷 size 50% 축소
- 전체 3연패: 당일 나머지 시간 신규 진입 중단
- 장중 cooldown: 30분 관찰 모드
- 손실 거래들의 공통점 자동 리포트

### 3-3. 시장 급락 시

한국 시장 안전장치:
- 서킷브레이커: 1단계 -8%, 2단계 -15%, 3단계 -20%
- 사이드카: KOSPI200 선물 -5% 이상 하락 1분 지속 시 프로그램 매도 5분 정지
- VI: 개별 종목 직전 체결가 대비 ±3~6% 변동 시 2분간 단일가 매매
- 단기과열종목: 3거래일 30분 단일가매매 (코스닥 예고 시 10거래일)

**액션:**
- KOSPI/KOSDAQ 지수 장중 -1.5%~-2% 이상이면 신규 long 축소/중지
- VI 다발, 서킷브레이커/사이드카 발생 시 safe mode
- 단기과열종목은 기본 제외
- 급락장일수록 "반등 먹기" 유혹은 초보에겐 거의 항상 독

### 3-4. 포지션 크기

**포지션 가치 = min(계좌리스크 기준, 최근 1분 거래대금의 5%, 상위 5호가 매도잔량 가치의 10%)**

추가 조건:
- spread > 3 ticks 또는 > 0.3%면 size 0
- 동시호가/VI/단기과열이면 size 0
- 거래대금만 많다고 안심 금지: 뉴스 급등주는 거래대금은 크지만 호가창은 얇은 경우 많음

## 4. 시스템 고도화 우선순위

### 4-1. 가장 먼저 개선할 것: 측정 체계

LLM 판단력이 아니라 측정 체계가 우선.

**액션:**
- 이벤트 원시 로그 정비: 원문 URL, 원문 시각, 수신 시각, 버킷, LLM 입력/출력, 주문, fill 가정
- 하드필터 선행: 유상증자, 공급계약 해지, VI, 단기과열, pre-move 급등, spread 급확대는 LLM 전에 잘라내기
- LLM은 middle zone만: 명백한 악재/호재는 deterministic rule
- LLM 출력 구조화: BUY/SKIP, reason_codes, confidence, expected_hold_window

### 4-2. 데이터 축적 단계별 분석

**3~5일치:** gross/net, bucket, market, 시총, 시간대, source latency, spread별 기초통계

**2~4주치:** 30초/1분/3분/5분/15분/30분/EOD forward return curve, MFE/MAE, 슬리피지, false negative 분석

**6~8주 또는 100건+:** walk-forward 검증, 버킷 pruning, hold-time 재설정

반드시 SKIP한 것도 같이 볼 것. 같은 데이터로 "버킷 선택"과 "보유시간 선택"을 동시에 하면 거의 반드시 과최적화.

### 4-3. 실전 전환 체크리스트

- [ ] 원문시각과 수신시각 저장 여부
- [ ] 계좌별 실제 수수료/세금 적용 여부
- [ ] 시장가 금지/허용 조건 정의 여부
- [ ] KRX/NXT 체결 가정 분리 여부
- [ ] 일일 손실한도/연속손실/시장 급락 kill switch
- [ ] VI/단기과열/거래정지/장후 공시 처리 규칙
- [ ] 최소 2주 이상 shadow run 수행 여부
- [ ] 룰 freeze 상태로의 OOS 결과 보유 여부

### 4-4. 사례 교훈

- Thomas Tarler: Claude/Cursor로 AI 트레이딩 앱 구축, 손절 모니터링 7일간 미작동으로 83개 포지션 누적. "진짜 병목은 전략이 아니라 운영 규율"
- Brett Harrison(전 FTX): "가장 수익성 높은 모델은 극도로 단순. LLM은 자율적 판단자가 아니라 증폭기로 사용해야"
- 한국 Quantus 사용자: CAGR 65%, MDD 30% 백테스트 → 6개월 실전 -20% 손실 (과적합)
- MDPI(2025): 시장 행동은 반응적이기보다 예측적. 순수 감성 분석보다 구조화된 이벤트 데이터가 더 강력한 신호

## 지금 당장 할 일 Top 5

1. 현재 룰과 프롬프트를 2주간 freeze, 모든 수정사항을 실험 장부에 기록
2. 원문시각~체결시각 전체 타임스탬프 체인 저장, KIS 수신 지연 측정
3. net PnL 엔진: 수수료·세금·슬리피지 반영해서 첫날 3건 다시 계산
4. 버킷 재설계: 체결/해지, 판매/구매, 소각/매입, 희석/비희석, 공식공시/언론기사 분리
5. 리스크 킬스위치: 일일 손실한도, 연속손실 중단, VI/단기과열/급락장 safe mode

## 절대 하지 말아야 할 것 Top 5

1. 첫날 3승 0패를 근거로 버킷이나 프롬프트를 강화하는 것
